# Session record: 2026-06-12

## Decision: NDE is semantic-native; LT-3 retired (operator-ratified)

The operator ratified two connected decisions in session:

1. NDE is a semantic solution. Its windowing, control, and input
   vocabulary is defined by NDE's own versioned contracts and grows
   by ADR. It is not derived from an imported toolkit's object
   model. Rationale: AppKit's meaning (responder chains, key
   windows, control roles) is toolkit-private and opaque to the
   system, which contradicts the substrate's thesis that intent is
   visible to the system. SDCS expresses drawing intent, semainput
   emits semantic gestures, chronofs makes time addressable; the
   desktop layer terminates that property in the user's hands rather
   than discarding it.

2. "GNUstep backend as a first-class citizen" is retired. LT-3
   closes as RETIRED (never started, no code removed). If GNUstep
   compatibility is ever wanted, it returns as a fenced compatibility
   tenant alongside NDE-5, under a new ADR, and must not influence
   the native semantic vocabulary.

Standing caution recorded with the ratification: a semantic NDE with
a fenced GNUstep tenant is coherent; a semantic NDE that quietly
inherits AppKit idioms because they are familiar is the worst of
both. The fence is the discipline.

What this does not change: D-7 (ADR 0011), D-10 (ADR 0012), and the
NDE-1 Milestone 1 plan are unaffected. Session-level enforcement
with surface-level semantics in D-7 was argued from the trust
boundary (getpeereid), not from AppKit compatibility, and stands.
LT-1 and LT-2 remain open as substrate mechanism serving the
semantic NDE.

## Addendum: semantic design ratified (same session)

The operator ratified the NDE semantic design
(NDE-SEMANTIC-DESIGN.md, destined for the NDE repository). The
design defines the node ontology, the five contracts (C1 surface
roles, C2 semantic tree, C3 presentation resolution, C4 input and
command, C5 canvas and temporal semantics), vocabulary governance,
and milestones M2 through M5 above the unchanged M1.

Decisions adopted with the ratification (per the in-line
recommendations): layout intent with resolver-computed geometry;
retained-tree mechanism in UTF (LT-1 grown with typed properties
and transactions) with vocabulary in NDE; bare fenced canvas at C2
v1 with temporal bindings at M5; tree-only non-canvas UI for native
applications. Open by the document's own terms: the v1 core
vocabulary (role by role) and the tree-read authority gate, both
taken in the C2 ADR series.

Consequences for this tree:

- The LT-1 review owed above is resolved in direction: LT-1 is
  designed once, as both the render-side layer carrier and the
  transport for typed properties and transactions. LT-1's entry
  should be revised to reflect this dual role before it opens
  (separate edit, not made in this session).
- CHN0002's deferred cross-process cursor-sharing decision now has
  its anticipated consumer (C5 temporal cursors); take it up when
  C5 opens, not before.
- M1 (NDE-1 via D-7) is explicitly unaffected; the current
  D-7 -> D-10 -> NDE-1 plan stands.

## Addendum 2: design reopened, Revision 2 issued (same session)

The operator reopened the ratified design with instruction to
correct where suboptimal. Revision 2 makes six corrections and
returns to DRAFT for re-ratification:

- A: the semantic tree is unbundled from LT-1 (revises adopted
  decision 2, now re-flagged). The tree encoding becomes a wire
  format and library in shared/, terminating at NDE; LT-1 stays
  pure presentation mechanism. The Revision 1 disposition is
  withdrawn as a design error (two structures with different
  shapes, cadences, and owners were fused).
- B: client-assigned stable node ids replace node paths for focus,
  relationships, and command addressing; commands carry tree
  generation and are never delivered to absent ids.
- C: resolver determinism restated in (tree, environment, chronofs
  instant); interaction state must live in the tree; authoritative
  timestamps applied by NDE at commit (single clock reader, total
  order per tree).
- D: C4 gains a normative text editing and composition
  sub-contract (preedit and conversion NDE-side, committed edits
  to the field); required for v1.
- E: virtualized collections (extent plus materialized window) for
  list, table, and grid.
- F: canvas raw-input escape scoped (focus-gated keyboard,
  bounds-gated pointer, beneath session enforcement and ADR 0012
  precedence).

Consequence withdrawn from Addendum 1: LT-1 is no longer designed
as a dual carrier; the LT-1 review resolves to presentation-only.
Decisions 1, 3, 4 remain adopted; decision 2 awaits
re-ratification; 5 and 6 remain open for the C2 ADR series.
